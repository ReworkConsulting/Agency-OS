# Agent Instructions

You're working inside the **WAT framework** (Workflows, Agents, Tools). This architecture separates concerns so that probabilistic AI handles reasoning while deterministic code handles execution. That separation is what makes this system reliable.

## The WAT Architecture

**Layer 1: Workflows (The Instructions)**
- Markdown SOPs stored in `workflows/`
- Each workflow defines the objective, required inputs, which tools to use, expected outputs, and how to handle edge cases
- Written in plain language, the same way you'd brief someone on your team

**Layer 2: Agents (The Decision-Maker)**
- This is your role. You're responsible for intelligent coordination.
- Read the relevant workflow, run tools in the correct sequence, handle failures gracefully, and ask clarifying questions when needed
- You connect intent to execution without trying to do everything yourself

**Layer 3: Tools (The Execution)**
- Firecrawl skills handle all web scraping and research — use these first
- Python scripts in `tools/` handle deterministic formatting and document assembly
- Credentials and API keys are stored in `.env`

**Why this matters:** When AI tries to handle every step directly, accuracy drops fast. By offloading execution to Firecrawl skills and deterministic scripts, you stay focused on orchestration and decision-making where you excel.

## Installed Firecrawl Skills

All web scraping and research runs through Firecrawl. Never write a Python scraper when a Firecrawl skill can do it.

| Skill | When to Use |
|---|---|
| `firecrawl-scrape` | Extract clean markdown from any URL (websites, social pages) |
| `firecrawl-browser` | JS-rendered pages that require a real browser (Google reviews, etc.) |
| `firecrawl-search` | Web search with full content extraction (Reddit, X, news, competitors) |
| `firecrawl-crawl` | Bulk extract every page on a site |
| `firecrawl-map` | Discover all URLs on a domain |
| `firecrawl-agent` | Structured JSON extraction from complex sites |
| `firecrawl-download` | Save a site as local markdown files |
| `firecrawl` | General-purpose — use when unsure which sub-skill fits |

## How to Operate

**1. Always start at the router**
When Christian sends a request, read `workflows/router.md` first. It tells you which workflow to run.

**2. Read the client folder before doing anything client-related**
Check `clients/{client-name}/overview.md` before any client work. Check `icp.md` to see if research is already done.

**2a. Supabase is the single source of truth for the Platform**
The web platform (`platform/`) reads client data exclusively from Supabase — it does not read markdown files. After any onboarding or data change:
- Run `python tools/sync_client_to_supabase.py <client-slug>` to push one client to Supabase
- Run `python tools/import_all_clients.py` to sync all clients at once
- The markdown files are the working copy; Supabase is what the Platform reads

If a client does not appear in the Platform UI, it's because they haven't been synced to Supabase yet.

**3. Read the workflow before executing it**
Workflows are your operating instructions. Follow them step by step. Do not improvise around them.

**4. Use Firecrawl before writing code**
For any scraping or web research: use the appropriate Firecrawl skill. Only fall back to Python if Firecrawl genuinely can't handle it.

**5. Learn and update when things fail**
- Read the full error
- Fix it and verify the fix works
- Update the workflow so it doesn't happen again

**6. Keep workflows current**
Do not create or overwrite workflows without asking unless explicitly told to. These are the operating instructions — they need to be preserved and intentional.

**7. Respect workflow dependency order**
Workflows have a required execution order. Never skip a prerequisite:
```
onboard_client → build_icp → generate_ads
                           → seo_audit
                           → generate_report
```
Always check that `icp.md` exists and is complete before running any generation workflow. If it doesn't exist, run `build_icp.md` first.

## Installing New Skills — Critical Rules

- **NEVER use `--all` flag** when installing skills. It installs for 41 AI tools and creates dozens of unexpected folders.
  - Wrong: `npx -y firecrawl-cli@latest init --all`
  - Right: `npx -y firecrawl-cli@latest init` (Claude Code only)
- After any install, run `ls -la` to check for unexpected folders. Delete anything that doesn't belong.
- Always add the API key to `.env` before running a tool that requires it.
- Node.js v18+ required. Installed via nvm (Node v20). Activate with:
  `export NVM_DIR="$HOME/.nvm" && [ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"`
